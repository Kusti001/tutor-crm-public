## **О проекте**
TutorCRM – это CRM-система созданная для учителей и репетиторов, предоставляющая условия для управления учениками, занятиями и учебным процессом.
## **Технологический стек**
**Backend:** Python, FastAPI (Asynchronous), SQLAlchemy ORM, Pydantic v2, JWT (PyJWT/Passlib)
**Database:** SQLite (локальная разработка / MVP) -> Миграция на PostgreSQL
**Frontend:** React, Tailwind CSS, shadcn/ui
**DevOps & Infrastructure:** Docker, Docker Compose
## **Основной функционал**
Подробное описание пользовательских сценариев и бизнес-логики со схемами взаимодействия доступно в файле: [USER_FLOW.md](docs/USER_FLOW.md)
### Безопасность и Доступ
* **JWT-аутентификация.
* **Защита данных:** Изоляция сущностей (преподаватель имеет доступ исключительно к своим ученикам, группам и расписанию).
### Students Management
* Полноценный CRUD-интерфейс для ведения базы учеников (добавление, редактирование, архивация).
* Гибкое профилирование: хранение контактных данных, мессенджеров, персональных заметок по обучению и текущего статуса обучения.
### Lessons Management
* **Индивидуальный и Групповой форматы:** Поддержка классических занятий "один-на-один" и групповых сессий.
* **Динамическая тарификация:** Возможность кастомизации стоимости одного и того же группового занятия для каждого участника отдельно (учет персональных скидок, пакетов или абонементов).
* **Трекинг посещаемости:** Фиксация статусов занятий (`Запланировано`, `Проведено`, `Отменено`) и статуса присутствия/оплаты каждого студента.
### Аналитический дашборд
* Агрегационный слой данных, собирающий ключевые метрики на одном экране: общая финансовая статистика, количество активных студентов, расписание ближайших занятий и задолженности по оплате.
## **Архитектура проекта**
При построении я старался следовать layered architecture. Компоненты проекта распределены по следующим слоям:
```
📁 app
├── 📁 api # PRESENTATION LAYER
│   ├── 📁 v1
│   │   ├── 📁 endpoints # Маршруты
│   │   │   ├── 📄 __init__.py
│   │   │   ├── 📄 auth.py
│   │   │   ├── 📄 dashboard.py
│   │   │   ├── 📄 lessons.py
│   │   │   └── 📄 students.py
│   │   ├── 📄 __init__.py
│   │   └── 📄 router.py # Главный роутер версии API
│   └── 📄 __init__.py
├── 📁 core # INFRASTRUCTURE LAYER (Config & Security)
│   ├── 📄 __init__.py
│   ├── 📄 config.py # Валидация настроек окружения через Pydantic Settings
│   ├── 📄 dependencies.py # Зависимости FastAPI
│   └── 📄 security.py # Логика хеширования паролей и генерации JWT
├── 📁 db # INFRASTRUCTURE LAYER (Database Engine)
│   ├── 📄 engine.py # Инициализация подключения к СУБД
│   ├── 📄 init_db.py
│   └── 📄 session.py # Настройка фабрики асинхронных сессий
├── 📁 models # DOMAIN LAYER (Database Entities)
│   ├── 📄 __init__.py
│   ├── 📄 base.py
│   ├── 📄 lesson.py
│   ├── 📄 lesson_participant.py
│   ├── 📄 student.py
│   └── 📄 user.py
├── 📁 repositories # INFRASTRUCTURE LAYER (Data Access Object)
│   ├── 📄 __init__.py
│   ├── 📄 lesson.py
│   ├── 📄 student.py
│   └── 📄 user.py
├── 📁 schemas # PRESENTATION LAYER
│   ├── 📄 __init__.py
│   ├── 📄 lesson.py
│   ├── 📄 student.py
│   └── 📄 user.py
├── 📁 services # APPLICATION LAYER (Business Logic)
│   ├── 📄 __init__.py
│   ├── 📄 dashboard.py
│   ├── 📄 lesson.py
│   ├── 📄 student.py
│   └── 📄 user.py
├── 📄 __init__.py
└── 📄 main.py # Точка входа в приложение, инициализация FastAPI приложения
```
файловая структура проекта, полученная с помощью моего [treeprinter](https://github.com/Kusti001/micro-utils/tree/main/treeprinter)
### 1. Presentation Layer (Слой представления)
   * **Компоненты:** `api/v1/endpoints/` (Эндпоинты FastAPI), `schemas/` (Pydantic-модели запросов и ответов).
   * **Обязанности:** Прием HTTP-запросов, валидация входных данных, сериализация ответов.
### 2. Application Layer (Слой приложения)
   * **Компоненты:** `services/` (Бизнес-логика).
   * **Обязанности:** Реализация сценариев использования приложения (Use Cases), координация выполнения бизнес-правил (например, репетитору нельзя создать занятие с чужим учеником).
### 3. Domain Layer (Доменный слой)
Подробное описание базы данных со схемами доступно в файле:[DB_SCHEME.md](app/models/DB_SCHEME.md)
   * **Компоненты:** `models/` (Сущности ORM).
   * **Обязанности:** Ядро системы. Определение структуры сущностей и реляционных связей.
### 4. Infrastructure Layer (Инфраструктурный слой)
   * **Компоненты:** `repositories/` , `/db`.
   * **Обязанности:** Взаимодействие с СУБД через SQLAlchemy ORM. Изолирует вышележащие слои приложения от особенностей работы SQLAlchemy ORM и конкретной СУБД. При необходимости смены базы данных или ORM-библиотеки слой бизнес-логики останется абсолютно неизменным. !!!
## Project Roadmap
### Этап 1 `[DONE]`
*   **[+] Система аутентификации и авторизации:**
    *   Реализация механизма JWT для безопасного разделения доступа.
    *   Хеширование паролей пользователей.
    *   Управление зависимостями (Dependencies) для защиты приватных эндпоинтов.
*   **[+] Проектирование базовой архитектуры БД:**
    *   Разработка схем и моделей для ключевых сущностей приложения.
    *   Реализация базовых CRUD-операций (создание, чтение, обновление, удаление) с использованием SQLAlchemy ORM / FastAPI.
### Этап 2 `[В разработке]`
*   **Архитектура групповых занятий (Разработка новой ветки):**
    *   Перепроектирование моделей БД для поддержки связей (Many-to-Many) между студентами, преподавателями и учебными группами.
*   **Агрегация данных для дашборда:**
    *   Разработка эндпоинта для сборки дашборда (ближайшие уроки, статистика занятий, статусы оплаты).
*   **Интеграция с Frontend:**
    *   Разделение фронтенд-части на независимые переиспользуемые компоненты.
    *   Организация взаимодействия между фронтендом и бэкендом через REST API.
### Этап 3: `[Планируется]`
*   **Контейнеризация приложения:**
    *   Написание `Dockerfile` и `docker-compose` (Бэк + Фронт + СУБД).
### Этап 4 `[Будущее]`
*   **Модуль асинхронных уведомлений:**
    *   Разработка изолированного микросервиса (планируется на **Go**) для отправки уведомлений (Telegram Bot API / Email).
*   **Модуль аналитики и отчетности:**
    *   Сбор расширенной статистики по доходам репетитора и эффективности за выбранные периоды.
